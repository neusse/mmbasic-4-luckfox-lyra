# WebMite/PicoMite Port Map

This map connects the desired PicoMite/WebMite-style surface to the current
Luckfox MMB4L implementation. It is a planning document, not a claim that every
listed WebMite feature is implemented.

| WebMite/PicoMite area | Reference files or docs | MMB4L/Luckfox files | Luckfox status | Next action |
| --- | --- | --- | --- | --- |
| Interpreter core | PicoMite V6 manual; MMB4L core parser/token tables | `mmb4l/src/core`, `patches/mmb4l/0016-*`, `0020-*`, `0023-*`, `0024-*`, `0029-*` | MMB4L core is the base. Added `ARRAY SET`, PicoCalc `MODE` no-op, `BEEP` no-op, `DRIVE` no-op, and continuation lines. | Add selected command-table compatibility features such as `/*`, `*/`, `CHAIN`, `LMID`, `BIT` assignment, `BYTE` assignment, and `FLAG` assignment. |
| Function token surface | PicoMite V6 manual; MMB4L function token table | `mmb4l/src/core/tokentbl.c`, `docs/picomite-v6-language-target.md` | Token space is constrained. Existing aliases such as `MM.FONTWIDTH` and `MM.FONTHEIGHT` are implemented. | Review `BIT(`, `BYTE(`, `FLAG(`, and `TRIM$(` one at a time before spending tokens. |
| Graphics/display | PicoMite/WebMite graphics commands; MMB4L graphics | `mmb4l/src/common/graphics.*`, `patches/mmb4l/0009-*`, `0011-*`, `0015-*`, `0017-*`, `0018-*`, `0021-*`, `0022-*`, `0025-*`, `0026-*`, `0027-*`, `0030-*`, `0031-*`, `0032-*`, `0033-*` | Native fbdev-backed 320x320 PicoCalc graphics work for the supported console/text-mode target. Many drawing, image, sprite, page, and framebuffer commands are covered by target tests. | Continue real-program testing and visual regression coverage; use VM-MMBasic `host_fb` design as the cleanup reference. |
| Console/input | PicoMite `INKEY$`, keyboard behavior, MMB4L console input | MMB4L console/event code; `patches/mmb4l/0034-*`, `0035-*` | Terminal input works for SSH/ADB style use. Physical-console graphics-mode input is handled through the PicoCalc evdev keyboard backend. | Add key-specific regressions as real programs expose missing mappings. |
| Game/visual extensions | Sprite/page/framebuffer portions of PicoMite/CMM2 behavior | `patches/mmb4l/0025-*`, `0027-*`, target graphics tests | Page write/copy, framebuffer basics, and sprite/page ID separation are implemented. | Test more real games and add focused regressions for any remaining sprite/page quirks. |
| CSUB | PicoMite `CSUB` and `MM.INFO(CALLTABLE)` behavior | `patches/mmb4l/0010-*`, CSUB target tests | ARM Luckfox CSUB execution works for core helpers and verified user CSUBs. Hardware call-table entries remain limited. | Expand only when matching Linux hardware backends exist. Keep unsupported entries explicit. |
| Networking/WiFi status | WebMite WiFi and WEB documentation | `patches/mmb4l/0012-*`, `0019-*`, `docs/luckfox-networking.md` | Linux-owned WiFi status, IP status, `WEB SCAN`, `WEB SCAN array%()`, read-only `OPTION WIFI`, no-op `WEB NTP`, and outbound `WEB REST GET/POST` are implemented. | Debug any manual `WEB SCAN` target quirks and keep Linux network ownership policy clear. |
| Inbound web server | WebMite TCP server and transmit commands | Future Linux socket backend | Not implemented yet. `OPTION WEB MESSAGES` is identified as verbosity control, not the receiver itself. | Implement `OPTION TCP SERVER PORT`, `OPTION WEB MESSAGES`, `WEB TCP INTERRUPT`, `WEB TCP READ/SEND/CLOSE`, `WEB TRANSMIT PAGE`, and `WEB TRANSMIT FILE`. |
| Raw TCP client | WebMite TCP client commands | Future Linux socket backend | Not implemented. REST covers the common HTTP/HTTPS API case. | Add raw TCP client commands only when compatibility demand justifies the work. |
| Audio | PicoMite/WebMite sound commands; MMB4L SDL audio | `patches/mmb4l/0023-*`, `0028-*`, `docs/picocalc-target.md` | `BEEP` is a no-op. `PLAY SOUND` can continue when audio open fails. Real PicoCalc audio is not complete. | Verify SDL audio over ALSA, then add direct ALSA only if needed. |
| Hardware IO | PicoMite GPIO/I2C/SPI commands; board pinout | Future `docs/picocalc-pin-map.md`; future Linux GPIO/I2C/SPI backends | Not implemented for real PicoCalc hardware. Safety map is required first. | Classify pins/buses before exposing GPIO, I2C, or SPI commands. |
| File/storage | MMBasic filesystem commands; Linux filesystem | MMB4L filesystem support; `patches/mmb4l/0024-*` | Linux paths are authoritative. `DRIVE` is accepted as a no-op for old programs. | Review `CHDIR`, `SAVE`, `LIBRARY`, and path dialect behavior with real SD card programs. |
| Checker/testing | Installed tests and compatibility scanning | `tools/mmb4l_check_basic.py`, `tests/`, `scripts/target/mmb4l-run-tests` | Target tests and static BASIC checker are included in the release bundle. | Improve include/library awareness, known-program profiles, and substitution suggestions. |
